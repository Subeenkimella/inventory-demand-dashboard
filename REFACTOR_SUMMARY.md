# 리팩터링 요약 (스토리라인 기준 전면 재구성)

## 삭제한 요소 목록

- **사이드바**: 예측 설정(모델/기간/학습 구간) 전부 제거 → 공통 필터(카테고리/창고/SKU/기준일)만 유지
- **탭**: 입출고 추적 탭, 재고 적정성 탭 제거 → 탭 4개만 사용
- **지표/기능**: MAPE·신뢰도, 예측 기준/실적 전환, 과잉 SKU 수 KPI, DOS 히스토그램·구간별 카드, 예외만 보기/추천 사유 필터
- **개요**: 품절 위험/과잉·무수요/오늘 조치 등 Top N 리스트 전부 제거 → KPI 4개 + 해석 문장 + "지금 가장 먼저 봐야 할 이유" 3줄만 유지
- **용어(금지어)**: 과잉, 무수요, 핫이슈, 오늘 조치, Top5/Top10, 조건 기반 리스트, 핫 SKU, 주요 SKU → 현업 표현(재고 과다 SKU, 최근 N일 수요가 없는 SKU, 우선 점검 대상, 즉시 발주 검토 대상, 재고 조정 검토 대상)으로 대체

---

## 각 탭이 스토리라인에서 맡는 역할 (1줄)

| 탭 | 역할 |
|----|------|
| **Overview (요약)** | 1번 질문(지금 재고 상태는 안전한가?)에 답하는 KPI 4개와 해석 문장, 그리고 "지금 가장 먼저 봐야 할 이유" 3줄로 상태 판단을 가능하게 함. |
| **재고 위험 원인 분석** | 2번 질문(어떤 SKU가 문제인가, 왜?)에 답하는 수요×DOS 매트릭스와, "재고 커버 일수가 정책 기준보다 짧고 수요 영향도가 높은 SKU" 리스트로 원인을 설명함. |
| **품절 발생 시점 분석** | 3번 질문(언제 문제가 터지는가?)에 답하는 예상 품절일·DOS·상태 마크(🔴긴급/🟠주의/🟢안정)로 시점을 명확히 함. |
| **권장 발주·재고 조정** | 4번 질문(무엇을 조치해야 하는가?)에 답하는 사유·조치하지 않을 경우 리스크·권장 조치(발주/재고 감축/재고 조정 검토) 테이블로 행동을 유도함. |

---

## 스토리라인 (변경 금지)

1. 지금 재고 상태는 안전한가? (Status)
2. 어떤 SKU가 문제인가, 왜 문제인가? (Cause)
3. 언제 문제가 발생하는가? (Time)
4. 그래서 무엇을 조치해야 하는가? (Action)
